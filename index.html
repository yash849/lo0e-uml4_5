UML4.5 Git & Use case diagram

Introductie

Doelstelling

  • Behendig zijn in: commits, branching, merging, pushing en pulling
  • Use case diagrammen kunnen maken

Startinstructies

  • Fork deze repository naar jouw eigen github account
  • Clone deze fork naar jouw computer
  • Open je favoriete git applicatie en editor.
  • Volg de instructies!!!!!

Afbeeldingen toevoegen

Opdracht

Voor je begint met het aanpassen van de code, zorg dat je in de develop branch zit.

  • Maak een nieuwe branch: afbeeldingen
  • Controleer dat je in deze branch zit
  • Vervang bovenaan de pagina "Naam van student" door jouw eigen naam (id=naam)
  • Commit deze wijziging
  • Zoek een passende afbeelding voor DIT hoofdstuk, voeg deze afbeelding toe in de map images en maak een commit.
  • Pas de src aan van de placeholder afbeelding van DIT hoofdstuk aan, zodat de nieuw toegevoegde afbeelding zichtbaar is en maak een commit.
  • Ga terug naar de develop branch.
  • Merge de aanpassingen van afbeeldigen naar develop.
  • Push de develop branch naar github

Use case diagram

  • Start een nieuwe branch use-case-diagram vanuit develop
  • Maak een Use Case Diagram voor de onderstaande User-stories
    Maak hierbij gebruik van:
    • Include relaties
    • Extend relaties
    • Primaire & secundaire actor(s)
  • Vervang onderstaande placholder image (id=use-case-image) met jouw use case diagram.
  • Commit deze aanpassing
  • Merge de aanpassingen van use-case-diagram naar de develop branch
  • Push de develop branch naar github

User stories

  • Als student wil ik inloggen zodat ik mijn studentenzaken kan regelen
  • Als student wil ik mijn cijfers kunnen bekijken, zodat ik weet hoe ik er voor sta
  • Als student wil ik mijn aanwezigheid kunnen zien, zodat ik weet of leerplicht mij gaat stalken.
  • Als student wil ik mijn rooster kunnen inzien, zodat ik geen lessen mis.
  • Als student wil ik mijn ouders toestemming kunnen geven om mijn cijfers te kunnen inzien, zodat ze ophouden met er naar vragen
  • Als ouder wil ik de cijfers kunnen zien van mijn minderjarige kind, zodat ik weet hoe hij/zij ervoor staat.
  • Als ouder wil ik de aanwezigheid kunnen zien van mijn minderjarige kind, zodat ik weet of ze wel op school zijn overdag. bestellen.
  • Als docent wil ik cijfers kunnen invoeren, zodat de voortgang van de studenten wordt bijgehouden.
  • Als docent wil ik de aan-/afwezigheid van mijn les kunnen invullen, zodat afwezige studenten worden opgemerkt.
  • Als leerplicht wil ik een seintje krijgen als een student afwezig wordt gemeld door docent, zodat ik er achteraan kan bellen.
